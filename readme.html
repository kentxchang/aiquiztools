前往「剛好測」軟體 By 軟體教學手冊生成

🚀 快速上手：開始您的即時測驗

「剛好測」是一個無需學生帳號的即時線上測驗系統。教師可透過 AI 輔助快速出題，學生輸入姓名即可作答，成績與分析即時呈現於教師儀表板。以下三步驟將引導您快速開始：

1
進入教師管理後台

進入首頁後，點擊「我是老師」，系統會要求輸入教師密碼 （預設為 tpet） 進行驗證。驗證通過後，您將進入「教師管理儀表板」。

2
建立您的測驗題庫

在儀表板點擊「新增題庫」按鈕，設定測驗標題、分類，並選擇題型（一般選擇題或閱讀測驗）。您可以利用內建的 AI 功能或手動匯入方式建立題目。

注意：AI 出題功能需要您事先在「系統設定」中配置有效的 Gemini API Key 才能使用。

3
啟用測驗並邀請學生

在題庫列表中，找到您想使用的測驗，點擊「啟用」按鈕。系統將在全球狀態中標記此測驗為進行中。學生在首頁點擊「我是學生」，輸入姓名後，即可自動載入您啟用的測驗並開始作答。

🧠 核心測驗模式：兩種主要評量類型

A. 一般選擇題 (Multiple-Choice)

適用於單一知識點的快速評量。支援 AI 依難度（簡單/一般/困難）自動生成題目，或透過手動/貼上方式快速建置題庫。

適用情境：課堂結束後的隨堂小考、單元重點複習測試、或從課本/講義直接擷取內容進行快速出題。

B. 閱讀測驗 (Reading Comprehension)

整合式測驗，支援學生作答時使用螢光筆畫記。教師可設定文本來源（貼上、上傳檔案或 AI 主題生成），並依 PIRLS 四層次結構生成題目。

適用情境：中長篇閱讀理解評量，特別針對國文或英文科目，並需要觀察學生「畫記」習慣作為分析參考。

✨ 進階功能與 AI 應用：高效出題與精準評量

AI 智能出題：多媒體來源支援

在建立「一般選擇題」時，您可以選擇「AI 自動出題」模式。AI 支援多種來源內容，實現一鍵出題：

  • 純文字貼上：直接貼上講義、課文或文章內容。
  • 圖片上傳：上傳筆記、黑板照片或圖表，AI 將會理解內容並根據其生成題目。
  • PDF 檔案解析：支援直接上傳 PDF 檔案，系統會自動解析出文字內容作為出題依據。

您可自訂「簡單題數」、「一般題數」和「困難題數」的比例，以控制測驗難度分佈。

閱讀測驗：PIRLS 層次設定

在建立「閱讀測驗」時，您可以精確控制 AI 依據 PIRLS 閱讀理解四層次出題的數量：

  • 提取訊息 (Retrieval)：考驗從文本中直接找出答案的能力。
  • 推論理解 (Inference)：考驗根據文本線索進行簡單推斷的能力。
  • 詮釋整合 (Interpretation)：考驗整合文本不同部分資訊，理解主旨的能力。
  • 評估鑑賞 (Evaluation)：考驗批判性思考，評估作者觀點或寫作手法的能力。

提示：您可以指定 AI 輸出語言 (繁中/英文/日文/韓文)，並可選擇讓 AI 依據您輸入的「主題」自動生成一篇閱讀文本。

題庫精細化管理

所有題庫都具備完整的編輯功能：

  • 手動/貼上：支援使用格式化文字（題目、答案、選項、解析）快速貼上並批量匯入題目。
  • 單題編輯：在題目列表中，您可以隨時對單一題目進行修改（標題、選項、答案、難易度、解析），甚至可以單獨刪除某一題。
  • 複製題庫：點擊「複製」按鈕，可以快速生成一個完全一樣的題庫副本，方便您進行版本修改或調整難度。

📊 測驗啟用與資料分析：掌握學習成效

測驗啟用與停止

  • 啟用：在儀表板點擊題庫上的「啟用」按鈕，即代表該測驗開始進行。此時學生端將立即顯示該測驗。
  • 停止：已啟用的測驗上方會出現綠色橫幅，點擊橫幅上的「結束測驗」按鈕，即可停止該測驗，學生將無法再進入作答。
  • 分類與篩選：您可以依據自訂的「分類」標籤，快速篩選儀表板上的題庫列表。

成績分析總覽

點擊題庫上的「查看分析」按鈕，可查看詳細的測驗報告：

  • 成績分佈：透過長條圖（0-20, 21-40...）直觀了解全班的成績分佈概況。
  • 學生列表：按分數高低排序的學生作答列表，可點擊「詳情」進入個別學生報告。
  • 下載報表：點擊「下載報表」按鈕，可匯出 CSV 格式的成績紀錄，包含學生姓名、得分、作答詳情，方便匯入成績系統。

試題分析與錯誤診斷

在分析報告中切換到「試題分析」頁籤：

  • 答對率：每道題目的答對率長條圖，讓您快速找出全班的弱點知識點（答對率低的題目）。
  • 答錯名單：每題皆可展開查看所有答錯的學生姓名，以及他們選擇的錯誤選項，利於點名檢討或個別輔導。

個別學生報告與畫記內容

在個別學生報告中，除了詳盡的答題正確性分析外，如果測驗類型為「閱讀測驗」，您還可以查看該學生在作答過程中使用的螢光筆畫記內容，作為評估其閱讀策略與專注度的重要參考。

❓ 常見問題

Q: 學生可以在作答過程中切換回首頁嗎？

A: 學生在作答時應保持在測驗畫面。如果學生不小心關閉網頁或切換畫面，只要重新開啟並輸入相同姓名，系統將會引導他回到目前的測驗進度（已作答的答案會被保留）。

Q: 忘記教師密碼怎麼辦？

A: 預設教師密碼為 tpet。此密碼是寫死在程式碼中的，目前無法從介面修改。如果您需要更改密碼，必須直接編輯 HTML 程式碼中的相應部分。

Q: 我可以同時啟用多個測驗嗎？

A: 為了避免學生混淆，「剛好測」在任何時間點只允許啟用一個測驗 (Active Quiz)。當您啟用新的測驗時，舊的測驗會自動停止。若您需要管理多個班級，請使用不同的瀏覽器或分頁，分別部署獨立的「剛好測」應用程式。

Q: 學生端看到的螢光筆畫記內容，老師也看得到嗎？

A: 是的。學生在「閱讀測驗」中使用的所有畫記（包含顏色和位置）都會與答案一起儲存到資料庫。教師在「成績分析」中查看學生報告時，可以同步看到該學生的畫記內容，作為分析其閱讀習慣的輔助資料。

🔧 建置系統與 API：設定持久化與 AI 功能

「剛好測」需要連接 Google Firebase Firestore 作為資料庫，以及 Gemini API Key 來啟用 AI 輔助出題功能。這兩項設定都可以在應用程式右上角的「齒輪圖示 (系統設定)」中完成。

注意：若未完成 Firebase 資料庫設定，系統將無法儲存任何題庫或作答紀錄。

1. Firebase 資料庫連線設定

您需要建立一個 Firebase 專案，並啟用 Authentication（匿名登入）和 Firestore Database。您可以點擊設定視窗中的「查看圖文教學」按鈕，按照詳細步驟操作。

  1. 在 Firebase Console 註冊 Web App，並取得 firebaseConfig 程式碼區塊。
  2. 將該程式碼區塊的內容貼到「系統設定」中的「資料庫連接」文字框。
  3. 點擊「儲存並連線」。系統會將設定儲存在您的瀏覽器中並進行初始化連線。
  4. **重要：**請確保您已設定 Firestore 安全性規則，允許匿名使用者讀寫資料，否則儲存操作將會失敗。

2. AI API Key 設定

AI 智能出題功能需要呼叫 Google Gemini API。您必須提供自己的 API Key。

  1. 前往 Google AI Studio 申請 Gemini API Key。
  2. 將取得的 Key 完整貼入「系統設定」中「AI API 設定」區塊的輸入框。
  3. 系統會自動儲存您的 Key 於瀏覽器中，並在您需要 AI 服務時自動使用。

提示：AI API Key 是獨立於 Firebase 設定的。即使您沒有 Firebase 資料庫，只要有 Key，AI 出題功能在儲存到本地題庫時仍可使用。

3. 資料庫免費額度與注意事項

「剛好測」基於 Firebase 免費方案運行。由於所有測驗資料都是文字或結構化數據，其儲存空間和讀寫次數的消耗極低，在正常教學情境下，您幾乎不用擔心會超過 Firebase 的免費額度。

總結：：對於純文字選擇題測驗來說，Firebase 免費方案非常充裕。您只需確保在 Firebase 設定中正確啟用了匿名驗證和 Firestore 資料庫即可。

🔗 部署與分享：多種方式分發應用程式

完成所有設定後，您可以透過多種方式將「剛好測」發佈給其他教師或學生使用。

1. 教師用分享連結 (含設定)

在「系統設定」的「分享應用程式」區塊中，可以找到「教師用分享」連結：

  • 功能：：此連結（或對應的 QR Code）包含了您的 Firebase 連線設定。
  • 用途：：供您在不同裝置間同步設定，或分享給其他協作教師，讓他們可以跳過 Firebase 設定步驟直接開始使用。

2. 學生專用分享連結 (簡化介面)

在「系統設定」中，可以找到「學生專用分享」連結：

  • 功能：：此連結會強制應用程式以「學生模式」啟動。
  • 效果：：學生打開後將直接進入學生登入畫面，完全隱藏教師入口、設定按鈕和所有管理功能。
  • 用途：：請將此連結或 QR Code 提供給您的學生，以避免他們誤觸或更改您的系統設定。

3. 靜態網頁空間部署

如果您不希望每次都從本地檔案開啟，建議將 HTML 檔案部署到靜態網頁空間（例如：GitHub Pages, Vercel, 或您學校的網頁伺服器）。

  1. 使用您喜歡的方式將 剛好測V2.html 檔案上傳到網頁空間。
  2. 學生和教師即可透過網址隨時存取。

提示：：部署完成後，您可以將「學生專用分享連結」設定為該網頁的書籤或投影給學生，方便他們快速登入。
